Checklist pour l’interface :

* Affichage de la carte :
[ ] La carte occupe la majeure partie de l'écran
[ ] N'afficher que les traces de la journée ou de la semaine
[ ] Permettre de cliquer directement sur un segment de la carte sans passer par la liste
[ ] Permettre de sélectionner plusieurs séquences sur la carte via un clic multiple
[ ] Utilisation de pictogrammes et de couleurs pour expliciter les données géographiques (ex : voiture → rouge, vélo → bleu)
[ ] Aucune animation ne se déclenche au chargement de la carte ; les tuiles sont chargées à la demande uniquement



* Filtres et navigation :
[ ] Afficher seulement quelques filtres standards par défaut ; les filtres avancés dans un panneau rétractable
[ ] Proposer un accès direct aux traces de la dernière session, sans avoir à filtrer
[ ] Situer clairement les traces dans leur contexte temporel (ex : "Aujourd'hui", "Cette semaine")
[ ] Chaque fonctionnalité principale est accessible en moins de 3 clics depuis l'écran d'accueil



* Listes et contenu affiché
[ ] La liste des traces n'affiche que les informations essentielles ; les détails sont disponibles au clic
[ ] Limiter le nombre d'éléments affichés dans les listes
[ ] Ne pas afficher de blocs de textes explicatifs permanents ; utiliser des tooltips pour les détails
[ ] Un bouton "Voir plus" explicite remplace tout chargement automatique en bas de liste ; aucun défilement infini



* Vocabulaire et représentation
[ ] Utiliser un vocabulaire proche du langage courant (ex : "Trajet" plutôt que "Séquence")
[ ] Afficher les données temporelles dans un format naturel (ex : "8h40", "Lundi 9 Mars")
[ ] Ajouter des pictogrammes pour une compréhension rapide (ex :  à côté du mode vélo)
[ ] Afficher des messages clairs et intelligibles (ex : "Déplacement modifié avec succès")
[ ] Seules les données du participant connecté sont affichées



* Efficacité et personnalisation
[ ] Permettre aux utilisateurs de nommer librement un déplacement ou un arrêt (ex : "École", "Trajet travail")
[ ] Proposer automatiquement le mode de déplacement d'après les données (ex : 30 km/h → Voiture)
[ ] Reconnaître automatiquement les POI récurrents et proposer à l'utilisateur de les compléter
[ ] Limiter la palette de couleurs dans l'interface et les légendes
[ ] L'interface est utilisable et lisible sur mobile (375px), tablette (768px) et desktop ; aucun élément ne déborde ou ne se superpose selon le terminal
